# HTML Semantic Elements

- In this part, I am learning about **HTML Semantic Elements**.  
- Semantic elements clearly describe their meaning to both the browser and the developer.  
- They help create well-structured, readable, and SEO-friendly web pages.

---

## What Are Semantic Elements?

Semantic HTML elements tell **what type of content** they contain.  
Unlike generic tags like `<div>` and `<span>`, semantic tags provide meaning.

---

## Common HTML Semantic Elements

### 1. `<header>`

```html
<header>
  <h1>My Website</h1>
  <p>Welcome to my webpage</p>
</header>
```

- Represents the top section of a page
- Usually contains logo, title, or navigation

---

### 2. `<nav>`

```html
<nav>
  <a href="#">Home</a> |
  <a href="#">About</a> |
  <a href="#">Contact</a>
</nav>
```

- Used for navigation links
- Helps search engines understand menus

---

### 3. `<section>`

```html
<section>
  <h2>About Me</h2>
  <p>This section contains information about me.</p>
</section>
```

- Groups related content
- Used to divide a page into sections

---

### 4. `<article>`

```html
<article>
  <h2>Blog Post</h2>
  <p>This is an article content.</p>
</article>
```

- Represents independent content
- Commonly used for blog posts, news, or cards

---

### 5. `<aside>`

```html
<aside>
  <p>This is sidebar content.</p>
</aside>
```

- Used for side content
- Examples: ads, related links, tips

## Semantic vs Non-Semantic Elements

| Non-Semantic | Semantic |
|--------------|----------|
| `<div>` | `<section>` |
| `<span>` | `<article>` |

- Semantic elements improve code readability
- Make websites more accessible and SEO-friendly

---

## Why Semantic HTML Is Important

- Improves accessibility for screen readers
- Helps search engines understand content
- Makes code easier to read and maintain
- Encourages best web development practices
